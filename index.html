📦 전체 백업 vs 증분 백업 비교

구분 전체 백업 (Full Backup) 증분 백업 (Incremental Backup)
백업 내용 전체 데이터를 매번 백업 변경된 부분만 백업
소요 시간 매번 오래 걸림 짧은 시간 소요
저장 공간 많은 공간 필요 적은 공간 사용
복구 속도 빠른 복구 여러 백업 조합 필요
관리 복잡도 간단함 복잡함
네트워크 부하 높음 낮음
비용 높음 낮음
📊 백업 용량 비교 (7일간)
⏱️ 백업 시간 비교
📋 실제 백업 시나리오 예시

상황: 100GB 데이터베이스, 매일 5GB씩 변경

월요일
전체: 100GB
증분: 100GB
화요일
전체: 105GB
증분: 5GB
수요일
전체: 110GB
증분: 5GB
목요일
전체: 115GB
증분: 5GB
금요일
전체: 120GB
증분: 5GB

결과:

  • 전체 백업 총 용량: 550GB (100+105+110+115+120)
  • 증분 백업 총 용량: 120GB (100+5+5+5+5)
  • 절약 용량: 430GB (78% 절약)

🟢 전체 백업의 장점

  • 단순하고 이해하기 쉬움
  • 복구 시 한 번에 완료
  • 백업 파일 하나로 전체 복구
  • 관리가 간편함
  • 데이터 일관성 보장

🟡 전체 백업의 단점

  • 많은 저장 공간 필요
  • 백업 시간이 오래 걸림
  • 네트워크 대역폭 많이 사용
  • 비용이 많이 듦
  • 시스템 부하가 큼

🟢 증분 백업의 장점

  • 저장 공간 효율적
  • 빠른 백업 속도
  • 네트워크 부하 적음
  • 비용 절약
  • 시스템 영향 최소화

🟡 증분 백업의 단점

  • 복구 시 여러 파일 필요
  • 관리가 복잡함
  • 백업 체인 손상 위험
  • 복구 시간이 오래 걸릴 수 있음
  • 의존성 관리 필요
